# Claude Rule (Git)

## 規約
- 指示がない限り、変更のコミットやプッシュをしない
- コミットメッセージは英語で、1行目は動詞の原形で始める (例: "add", "fix", "update")
- `--force-push` や `reset --hard` は事前確認なしに実行しない

## シークレット管理
- `.env`、クレデンシャル、秘密鍵を含むファイルは絶対にコミットしない
- シークレットをコードに直書きしない。パラメータストア(SSM)、Secrets Manager、環境変数を使う
- コード内にハードコードされたシークレットを検出した場合は指摘し、代替手段を提案する
